# User Management Guide

This guide covers how to manage users and their data in your application.

## Getting User Information

### Current User

Access the currently authenticated user:

```typescript
import { onAuthStateChanged } from 'firebase/auth';

onAuthStateChanged(auth, (user) => {
  if (user) {
    console.log('User is signed in:', user.uid);
    console.log('Email:', user.email);
    console.log('Display name:', user.displayName);
  } else {
    console.log('User is signed out');
  }
});
```

### User Properties

The user object contains useful information:

- `uid`: Unique user identifier
- `email`: User's email address
- `displayName`: User's display name
- `photoURL`: User's profile photo URL
- `emailVerified`: Whether email is verified
- `phoneNumber`: User's phone number (if provided)

## Updating User Profile

```typescript
import { updateProfile } from 'firebase/auth';

async function updateUserProfile(displayName: string, photoURL: string) {
  const user = auth.currentUser;
  
  if (user) {
    try {
      await updateProfile(user, {
        displayName,
        photoURL
      });
      console.log('Profile updated successfully');
    } catch (error) {
      console.error('Error updating profile:', error);
    }
  }
}
```

```typescript

  const metadata = user.metadata;

  console.log('Last sign in:', metadata.lastSignInTime);

```

## Best Practices

1. **Always Check User State**: Use `onAuthStateChanged` to track authentication state
2. **Verify Email**: Encourage users to verify their email addresses
3. **Handle Errors**: Properly handle errors when updating user information
4. **Re-authentication**: For sensitive operations, require re-authentication
5. **Secure Data**: Never store sensitive information in user profiles
